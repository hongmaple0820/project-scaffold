# 贡献指南

感谢你的贡献！

---

## 开发流程

1. Fork 仓库
2. 创建 feature 分支
3. 遵循认知工作流开发
4. 提交 PR

---

## 提交规范

```
类型(范围): 描述

[可选正文]

[可选脚注]
```

类型:
- `feat`: 新功能
- `fix`: Bug修复
- `docs`: 文档
- `style`: 格式
- `refactor`: 重构
- `perf`: 性能
- `test`: 测试
- `chore`: 构建

---

## PR 流程

1. 更新文档
2. 通过所有门控
3. 请求审查
4. 合并到 develop

---

## 代码审查

- 至少 1 人审查
- 解决所有评论
- CI 必须通过
