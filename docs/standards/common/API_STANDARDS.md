# API 规范

**版本**: v1.0  
**适用范围**: RESTful API

---

## 1. 响应格式

### 1.1 统一结构

```json
{
  "code": 200,
  "msg": "success",
  "data": {}
}
```

### 1.2 错误响应

```json
{
  "code": 3001,
  "msg": "用户不存在",
  "data": null
}
```

### 1.3 分页响应

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "records": [],
    "total": 100,
    "size": 20,
    "pages": 5,
    "current": 1
  }
}
```

---

## 2. 错误码

### 2.1 分段定义

| 范围 | 类型 |
|------|------|
| 200 | 成功 |
| 1000-1999 | 系统错误 |
| 2000-2999 | 认证授权 |
| 3000-3999 | 业务错误 |
| 4000-4999 | 第三方服务 |

### 2.2 常见错误码

| 码 | 含义 |
|----|------|
| 1000 | 内部错误 |
| 1001 | 数据库错误 |
| 1002 | 缓存错误 |
| 2000 | 未认证 |
| 2001 | 无权限 |
| 2002 | Token过期 |
| 3000 | 参数错误 |
| 3001 | 资源不存在 |
| 3002 | 资源已存在 |
| 3003 | 操作不允许 |

---

## 3. HTTP 状态码

| 状态码 | 使用场景 |
|--------|----------|
| 200 | GET/PUT/PATCH 成功 |
| 201 | POST 创建成功 |
| 204 | DELETE 成功（无返回） |
| 400 | 请求参数错误 |
| 401 | 未认证 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 409 | 资源冲突 |
| 429 | 请求过多 |
| 500 | 服务器内部错误 |

---

## 4. 请求规范

### 4.1 请求头

```
Content-Type: application/json
Authorization: Bearer {token}
X-Request-ID: {uuid}
X-Client-Version: 1.0.0
```

### 4.2 请求体

```json
{
  "name": "张三",
  "email": "zhangsan@example.com"
}
```

### 4.3 时间格式

ISO 8601: `2026-04-28T10:30:00+08:00`

---

## 5. 版本控制

### 5.1 URL 版本

```
/v1/users
/v2/users
```

### 5.2 Header 版本

```
Accept: application/json;version=1.0
```
