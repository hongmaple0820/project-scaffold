# 文档规范

**版本**: v1.0  
**适用范围**: 所有项目文档

---

## 1. 文档体系

```
docs/
├── README.md              # 文档入口
├── standards/             # 规范文档
├── guides/                # 开发指南
├── skills/                # 技能沉淀
├── architecture/          # 架构文档
└── plans/                 # 实施计划
```

---

## 2. Markdown 规范

### 2.1 文件格式

- UTF-8 编码
- LF 换行符
- 小写文件名
- 连字符分隔

### 2.2 标题层级

```markdown
# 一级标题（文档标题，仅一个）
## 二级标题（主要章节）
### 三级标题（子章节）
#### 四级标题（少用）
```

### 2.3 文档头部

```markdown
# 文档标题

**版本**: v1.0  
**日期**: 2026-04-28  
**作者**: 名字  
**状态**: 草稿/评审中/已发布

---
```

---

## 3. 文档类型

### 3.1 规范文档

- 清晰的 MUST/SHOULD/MAY 分级
- 具体可执行的规则
- 示例代码

### 3.2 指南文档

- 步骤说明
- 图文并茂
- 常见问题

### 3.3 技能文档

- 问题场景
- 解决方案
- 代码示例
- 注意事项

---

## 4. 文件命名

| 类型 | 命名格式 |
|------|----------|
| 规范 | `TOPIC-standards.md` |
| 指南 | `purpose-guide.md` |
| 技能 | `topic-patterns.md` |
| ADR | `ADR-NNN-title.md` |
| 计划 | `YYYY-MM-DD-topic/` |

## 5. 链接引用

### 5.1 内部链接

使用相对路径:
```markdown
[数据库规范](../standards/DATABASE_STANDARDS.md)
```

### 5.2 外部链接

标注来源:
```markdown
[官方文档](https://example.com) (2026-04-28访问)
```

## 6. 版本管理

### 6.1 版本号

```
v主版本.次版本.修订号

v1.0.0    # 正式发布
v1.1.0    # 新增内容
v1.1.1    # 修正错误
v2.0.0    # 重大重构
```

### 6.2 变更记录

```markdown
## 变更历史

| 版本 | 日期 | 作者 | 变更 |
|------|------|------|------|
| v1.1 | 2026-04-28 | xxx | 新增xxx |
| v1.0 | 2026-04-01 | xxx | 初始版本 |
```

---

## 7. 文档评审

### 7.1 评审检查

- 是否与现有规范冲突
- 是否清晰可执行
- 是否覆盖主要场景

### 7.2 状态标记

```markdown
**状态**: 草稿 → 评审中 → 已发布 → 已归档
```
