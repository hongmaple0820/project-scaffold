# 文档规范

**版本**: v1.1
**适用范围**: 所有项目文档

---

## 1. 文档体系

```
docs/
├── README.md              # 文档入口
├── standards/             # 规范文档
├── guides/                # 开发指南
├── skills/                # 技能沉淀
├── architecture/          # 架构文档
├── modules/               # 模块长期文档
├── adr/                   # 架构决策记录
├── workflow/              # 工作流说明和模板
└── worklog/               # 任务产物、验证和指标
```

---

## 2. Markdown 规范

### 2.1 文件格式

- UTF-8 编码
- LF 换行符
- 小写文件名
- 连字符分隔

### 2.2 标题层级

```markdown
# 一级标题（文档标题，仅一个）
## 二级标题（主要章节）
### 三级标题（子章节）
#### 四级标题（少用）
```

### 2.3 文档头部

```markdown
# 文档标题

**版本**: v1.0  
**日期**: 2026-04-28  
**作者**: 名字  
**状态**: 草稿/评审中/已发布

---
```

## 3. 文档类型

### 3.1 规范文档

- 清晰的 MUST/SHOULD/MAY 分级
- 具体可执行的规则
- 示例代码

### 3.2 指南文档

- 步骤说明
- 图文并茂
- 常见问题

### 3.3 技能文档

- 问题场景
- 解决方案
- 代码示例
- 注意事项

## 4. 模块化文档资产

文档必须按模块归属维护，不能把所有内容堆在一个总文档里。总文档只做入口、导航和关键红线，细节放到对应模块。

| 模块 | 目录 | 维护内容 |
|------|------|----------|
| 总入口 | `README.md`、`docs/README.md` | 项目定位、快速入口、核心索引 |
| Agent 规则 | `AGENTS.md`、`CLAUDE.md` | 工作方式、红线、完成定义 |
| 通用规范 | `docs/standards/common/` | 跨项目可复用规则 |
| 项目差异 | `docs/standards/projects/` | 业务项目特有规则和例外 |
| 架构 | `docs/architecture/` | 模块关系、边界、依赖、数据流 |
| 模块 | `docs/modules/<module>/` | README、API、数据、流程、风险 |
| ADR | `docs/adr/` | 长期有效的架构决策和取舍 |
| 工作流 | `docs/workflow/`、`scripts/workflow/` | 任务生命周期和脚本说明 |
| 任务记录 | `.planning/tasks/` | 探索、计划、验证、复盘 |
| 经验沉淀 | `thoughts/` 或项目约定目录 | 错误、复盘、资源索引 |

### 4.1 关联关系

- 新增或调整模块时，必须同步更新模块索引、架构图或模块关系说明。
- 一个事实只能有一个主来源；其他文档用链接引用，不复制粘贴。
- 文档引用代码、脚本、配置或接口时，必须写相对路径，并确保目标文件存在。
- 删除或移动文档时，必须更新所有入口索引和引用链接。
- 跨模块规则要写清上游、下游、依赖、影响范围和冲突处理方式。

### 4.2 资产产生与维护

| 变更 | 必须检查的文档资产 |
|------|--------------------|
| 新模块/新服务 | README、架构、项目规范、运行命令 |
| 新 API | API 规范、接口文档、调用方说明 |
| 数据库变更 | 数据库规范、迁移说明、回滚方案 |
| 配置/环境变更 | 部署规范、环境变量、密钥说明 |
| 工作流/门禁变更 | AGENTS/CLAUDE、workflow、gates 脚本说明 |
| Bug 修复 | worklog、错误知识库或复盘 |

### 4.3 版本与冲突

- 规范类文档更新必须调整版本号或变更历史。
- 同一模块多人协作时，优先拆分小文档，减少长文件冲突。
- 文档冲突不能只保留一侧内容，必须合并双方意图并校验链接。
- 文档和代码不一致时，以实际代码和验证结果为准，同时修正文档。
- 临时方案必须标明退出条件和负责人，不得长期留在入口文档。

## 7. 版本管理

### 7.1 版本号

```
v主版本.次版本.修订号

v1.0.0    # 正式发布
v1.1.0    # 新增内容
v1.1.1    # 修正错误
v2.0.0    # 重大重构
```

### 7.2 变更记录

```markdown
## 变更历史

| 版本 | 日期 | 作者 | 变更 |
|------|------|------|------|
| v1.1 | 2026-04-28 | xxx | 新增xxx |
| v1.0 | 2026-04-01 | xxx | 初始版本 |
```

## 8. 文档评审

### 8.1 评审检查

- 是否与现有规范冲突
- 是否清晰可执行
- 是否覆盖主要场景
- 是否按模块归档，并更新相关索引
- 是否处理了上下游文档和代码引用

### 8.2 状态标记

```markdown
**状态**: 草稿 → 评审中 → 已发布 → 已归档
```
