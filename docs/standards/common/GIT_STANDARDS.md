# Git 规范

**版本**: v1.0  
**适用范围**: 所有项目

---

## 1. 分支模型

### 1.1 分支类型

| 分支 | 用途 | 生命周期 |
|------|------|----------|
| `main` | 生产分支 | 永久 |
| `develop` | 开发分支 | 永久 |
| `feature/*` | 功能开发 | 临时 |
| `fix/*` | Bug修复 | 临时 |
| `hotfix/*` | 紧急修复 | 临时 |
| `release/*` | 发布准备 | 临时 |

### 1.2 流程

```
feature/x → develop → release/v1.0 → main
     ↑                    ↑
   fix/bug ───────────────┘
```

## 2. 提交信息

### 2.1 格式

```
类型(范围): 描述

[可选正文]

[可选脚注]
```

### 2.2 类型

| 类型 | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | Bug修复 |
| `docs` | 文档更新 |
| `style` | 代码格式 |
| `refactor` | 重构 |
| `perf` | 性能优化 |
| `test` | 测试相关 |
| `chore` | 构建/工具 |
| `ci` | CI/CD配置 |

### 2.3 示例

```
feat(auth): add OAuth2 login

- Support Google and GitHub OAuth
- Add token refresh mechanism
- Update login UI

Closes #123
```

## 3. PR 流程

### 3.1 PR 模板

```markdown
## 变更描述
简要描述本次变更

## 变更类型
- [ ] 新功能
- [ ] Bug修复
- [ ] 文档
- [ ] 重构

## 测试情况
- [ ] 本地测试通过
- [ ] 单元测试覆盖
- [ ] 集成测试通过

## 审查清单
- [ ] 代码符合规范
- [ ] 无安全漏洞
- [ ] 文档已更新
```

### 3.2 审查要求

- 至少 1 人审查
- 核心模块需 2 人
- CI 必须通过
- 解决所有评论

## 4. 版本标签

### 4.1 语义化版本

```
v主版本.次版本.修订号

v1.0.0    # 初始发布
v1.1.0    # 新增功能
v1.1.1    # Bug修复
v2.0.0    # 不兼容变更
```

### 4.2 创建标签

```bash
git tag -a v1.0.0 -m "Release v1.0.0"
git push origin v1.0.0
```
